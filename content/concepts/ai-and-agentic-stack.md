## Summary
This is the parent organizing layer for the wiki’s AI complex. It groups together the physical bottlenecks, deployment economics, and software-utility layers that sit between foundational model demand and real-world agent usage.

## Why this second-layer bucket exists
- Several existing concept pages describe different slices of the same stack: packaging, materials, optics, power, endpoint inference, and agentic utility rails.
- Without a parent layer, the wiki can feel like a list of adjacent trades rather than one connected architecture.
- This page separates AI infrastructure bottlenecks from other macro or consumer trades and makes the stack easier to navigate top-down.

- Deployment economics: [[inference-on-device]], [[all-chips-go-to-heaven]]
- Agentic software / usage layer: [[agentic-utilities]], [[bullshit-jobs]]

## Boundary rules
- Use this bucket when the core question is where AI value accrues across compute, bandwidth, power, device inference, or always-on agent workflows.
- Use [[strategic-industrial-capacity]] instead when the main thesis is national industrial scarcity, shipyards, transport, or physical strategic capacity outside the core AI stack.
- Use [[consumer-wallet-and-attention]] when the thesis is primarily about household demand, cash-flow stress, or cultural spending behavior.
